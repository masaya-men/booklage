# Booklage リビルド・ビジョン

> 作成: 2026-04-19 / ユーザーと深い対話で抽出した本気のリビルド方針
> **次セッションの Claude は、着手前に必ず全文読むこと。**

---

## 背景

2026-04 時点、Booklage は MVP として機能するが、ユーザー（非エンジニア・1人開発・生活をかけている）が「デザイン・体験の根本が違う」と感じている。競合（Raindrop、mymind、Cosmos、Are.na、Dewey、個人開発者の Twitter→Notion ツール等）が出揃った中で、**コラージュ表現と体験の気持ちよさ**で突き抜けない限り勝てない。

機能追加ではなく **パーツごとの作り直し** を選択する。

---

## 優先順位（ユーザー指定）

1. **B. ボード（2D flat の根本見直し）** — 最長滞在画面。完成度で全体印象が決まる
2. **C. LP / マーケティングページ** — 初見の印象。差別化の入口
3. **D. カードスタイル（Glass/Polaroid/Newspaper/Magnet）** — 「表現ツール」の核
4. **A. 3D 球体モード** — 既に master に opt-in で入ってる。polish は後回し

---

## リビルド原則（全セクション共通）

### やる
- **「触って気持ちいい」まで持っていくことが正義**。納期よりクオリティ
- 作り直す前に **「何が不満か」を言語化する**ブレストを必ず行う
- スプリング物理、proximity hover、イージングに神経を使う（Fluid Functionalism 思想）
- HTML-in-Canvas（VFX-JS）が使えるようになった今、カード量と表現の両立が可能
- ガラス・装飾はあとから載せる。まず骨組みで「超サクサク動く」状態を作る

### やらない
- 青→紫グラデーション、shadcn デフォルト、よくある SaaS っぽい UI
- Tailwind（現行通り Vanilla CSS + CSS Custom Properties）
- Framer Motion（GSAP + CSS keyframes）
- サーバー保存、ユーザー登録、ログイン

## セクション別ビジョン

### B. ボード（最優先）

**今の問題**:
- 無限キャンバス（2D 無制限）が本当に必要か疑問
- マウスホイールがズームなのは感覚に合わない（スクロール動作が自然）
- カードの自由サイズ変更が感覚的に動かない
- ブクマが溜まった後、フォルダ分け・既読管理のフローが欠けている

**方向性候補**（次セッションで選ぶ）:

1. **縦だけ無限**（デフォルト）
   - マウスホイール = 縦スクロール（自然）
   - カードは縦に流れる、Pinterest 的だがカスタム配置可能
   - 横は画面幅固定

2. **横だけ無限**
   - 時系列や展示用途向け
   - ギャラリー的体験

3. **背景テーマによって切替**
   - 例: 「ノート」= 縦、「カッティングマット」= 2D 無制限、「宇宙」= 3D 球体

**カードリサイズ作り直し**:
- 現行の GSAP Draggable + ガラス装飾を一旦全部外す
- まず骨組み（`<div>` のみ）で超サクサク動くドラッグ・リサイズを完成
- その上に装飾を載せ直す
- ハンドルの感触、スナップ、制約感にこだわる

**ブクマ管理フロー**:
- 右クリックで「既読」扱いにする
- 既読専用の保管エリア（無限キャンバス or 別ビュー）に自動移動
- フォルダ分けは **ドラッグ＆ドロップ + 候補サジェスト** で楽しく
- 大量ブクマを素早く処理できる "Inbox" 的モード

**ブックマーク名の自動取得強化**:
- YouTube → 動画タイトル（oEmbed or API）
- Twitter/X → ツイート冒頭 80 文字
- Instagram → 投稿本文冒頭
- 一般サイト → og:title フォールバック
- **全件、後から編集可能**

### C. LP / マーケティングページ

**今の問題**:
- 今のままで差別化できてる気がしない
- AdSense 申請用ページ `/features` `/guide` が未実装（戦略メモ）

**方向性候補**:

1. **Apple 風スクロール動画**（Safinaz Elhadary の手法）
   - GROK で scroll-optimized video 生成
   - v0 でフレーム分割 → scroll-scrubbing
   - Booklage のコラージュを動画で見せつける

2. **インタラクティブデモ**
   - LP 内で実際にカードを配置・コラージュできる
   - 触ったら離れられない

3. **動くコラージュ背景**
   - Scenery.io 的な常時動くビジュアル
   - VFX-JS の HTML-in-Canvas で大量カードを動かす

実装は次セッションのブレストで決める。

---

### D. カードスタイル

**今の問題**:
- 4 種類（Glass/Polaroid/Newspaper/Magnet）で「表現ツール」を名乗るには足りない？
- 各スタイルの完成度に差がある

**方向性候補**:
- 既存 4 種を作り直して全部高完成度に
- カードスタイル追加: ネオン、布、羊皮紙、切り抜き画用紙、プリクラ、ステッカー等
- スタイル × 背景テーマの相性マトリクス（Moodboard 的）

## 次セッション着手時の進め方

**新しい worktree を切って、B（ボード）から開始。**

手順:
1. `superpowers:brainstorming` スキルで「今のボードの何が不満か」を言語化
2. 方向性（縦無限 / 横無限 / テーマ切替）を決める
3. 骨組み（装飾無しのドラッグ・リサイズ）から作り直す
4. 既存の Canvas.tsx を捨てる決断も視野に
5. 触って気持ちよくなるまで polish
6. master にマージ → worktree 削除（クリア終了）

C（LP）は B がひと段落してから別セッションで。
