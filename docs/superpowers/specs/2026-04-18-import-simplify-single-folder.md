# 一括インポート仕様変更 — 単一フォルダ集約

> 作成日: 2026-04-18
> ステータス: 承認済み（ユーザー確認済み）
> 関連: `docs/superpowers/specs/2026-04-15-s9-bulk-import.md`（既存仕様）

---

## 背景

既存の一括インポート機能には2つの問題がある。

### 問題1: インポート後に「何も追加されない」ように見える

YouTube Watch Later CSV（140件）をインポートすると「140件追加しました！」と表示されるが、ボード上には何も現れない。ハードリロードすると新しいフォルダに追加されているのが見える。

**原因:**

1. パーサーが CSV 1行目を playlist 名として採用し、`bm.folder` にセット（`lib/import/parse-youtube-takeout.ts:14`）
2. `executeImport` がソースフォルダ名で新フォルダを作成（`lib/import/batch-import.ts:82-91`）
3. `handleImportComplete` が**新フォルダ一覧を再読込せず**、**画面遷移もしない**（`app/(app)/board/board-client.tsx:537-556`）
4. 結果: 保存は成功しているが、ユーザーの現在フォルダには何もなく、サイドバーにも新フォルダが現れない

### 問題2: 設計思想のズレ

AllMarks は「整理ツール」ではなく「**表現ツール**」。インポート元のフォルダ階層をそのまま持ち込む設計は、この思想と合わない。ユーザーが最初から Chrome の整理構造を AllMarks に移植したいわけではなく、「溜めていたブックマークをドサッと取り込んで、あとから自分で気に入ったものだけ整理・表現する」のが正しい UX。

---

## 変更の核

**パーサーが検出したフォルダ構造を無視し、全ブックマークを `インポート YYYY-MM-DD` という単一フォルダにまとめる。同日再インポートは同フォルダに追加。インポート完了時に自動で当フォルダへ遷移する。**

---

## 設計

### フォルダ生成ルール

- フォルダ名: `インポート YYYY-MM-DD`（ローカル日付）
  - 例: `インポート 2026-04-18`
- **同日再インポート時**: 既存の当日フォルダに追加（新規作成しない）
- **日付をまたぐ再インポート時**: 新しい日付でフォルダを新規作成

### データフロー変更

| ステップ | 現状 | 変更後 |
|---------|------|--------|
| パーサー群（`parse-*.ts`） | `bm.folder` にソースのフォルダ名を詰める | **変更なし**。パーサー出力の `folder` フィールドは後段で無視される |
| `buildFolderAssignments` | ソースフォルダ別に割り当てを生成 | **廃止**。代わりに「当日インポートフォルダを find-or-create」する単一関数に置換 |
| `executeImport` | folderAssignments に従って複数フォルダ作成 | 当日フォルダ ID を 1 つ確定し、全ブックマーク・カードをそこへ保存 |
| `handleImportComplete` | 現在フォルダの items のみ再読込 | **folders 一覧再読込 + 当日フォルダへ `setCurrentFolder` で自動遷移 + その items 読込** |

### UI 変更

ImportModal の 3 ステップ構成は維持する。ステップ 3（プレビュー）のみ簡素化。

**削除:**

- フォルダ割り当て UI（どのソースフォルダをどの AllMarks フォルダに、のドロップダウン群）
- 「全部このフォルダに」一括変更オプション
- `FolderAssignment[]` 状態管理

**維持:**

- 検出件数表示（「247件のブックマークを検出」）
- 重複スキップ件数表示（「12件が既存と重複のためスキップ」）
- 個別ブックマークの一覧プレビュー（URL + タイトル）

**追加:**

- 「`インポート 2026-04-18` に追加されます」という告知テキスト（実行ボタン直上）
  - 当日フォルダが既存なら「既存の `インポート 2026-04-18` に追加されます（現在 N 件）」
  - 新規作成なら「新しいフォルダ `インポート 2026-04-18` を作成して追加します」

結果、ユーザーは「検出 → 確認 → 実行」の 3 クリックで完了できる。

### 影響ファイル

| ファイル | 変更内容 |
|---------|---------|
| `lib/import/batch-import.ts` | `buildFolderAssignments` 廃止、`executeImport` を単一フォルダ版に書き換え、`findOrCreateImportFolder(db, date)` を新規追加 |
| `lib/import/types.ts` | `FolderAssignment` 型は import 内部で不使用になるため削除（export は残さない） |
| `components/import/ImportPreview.tsx` | 割り当て UI 削除、告知テキスト追加、`onExecute` のシグネチャから `assignments` 削除 |
| `components/import/ImportModal.tsx` | `onExecute` の型に追従 |
| `app/(app)/board/board-client.tsx` | `handleImportComplete` で folders 再読込 + 当日フォルダへ `setCurrentFolder` |
| `lib/import/parse-*.ts` | **変更なし**（`folder` フィールドは出力し続けるが、後段で無視される） |

### エッジケース

| ケース | 挙動 |
|--------|------|
| 同日再インポート | 既存 `インポート 2026-04-18` に追加。新規作成しない |
| 日付をまたいだ再インポート | 新フォルダ `インポート 2026-04-19` を作成 |
| 重複 URL（既存ブックマーク） | 既存の `deduplicate.ts` ロジックで除外（挙動変更なし） |
| 重複 URL（同一インポート内） | 既存の `deduplicate.ts` ロジックで除外（挙動変更なし） |
| YouTube Watch Later CSV のヘッダ行問題 | 自動解決（`bm.folder` を使わなくなるため影響なし） |
| ユーザーがインポート中にフォルダ作成/切り替え | インポート完了時に強制的に当日フォルダへ遷移 |
| ユーザーに既存フォルダが1つも無い状態でインポート | 当日フォルダが最初のフォルダになる。サイドバー表示 OK |
| タイムゾーン | ユーザーの**ローカル日付**を使う（`new Date().toLocaleDateString('sv-SE')` で `YYYY-MM-DD` 形式）。UTC ではなくローカルにする理由: ユーザーの感覚「今日入れた」に合わせるため |

### 残すべき挙動（重要）

- **パーサー出力の `bm.folder` フィールドは削除しない**。理由: 将来「ソース別フォルダ分け」をオプションとして提供する可能性、および既存パーサーへの影響を最小化するため。`ImportedBookmark.folder` 型は optional のまま残す
- **OGP 取得のバックグラウンド処理**は変更なし
- **プログレス表示**は変更なし（`phase: 'saving' | 'ogp'`）
- **重複判定（`deduplicate.ts`）**は変更なし

## テスト方針

### 既存テストの修正

- `buildFolderAssignments` のテスト削除
- `executeImport` のテストを単一フォルダ前提に書き換え
- `ImportPreview` のスナップショット/DOM テストを更新

### 追加テスト

| # | テスト内容 | 期待結果 |
|---|-----------|---------|
| 1 | 初回インポートで当日フォルダが新規作成される | `インポート 2026-04-18` が `folders` に追加 |
| 2 | 同日内の 2 回目インポートで既存フォルダに追加される | フォルダ数が増えず、bookmark 数が合計される |
| 3 | 日付をまたいだインポートで新フォルダが作られる | `インポート 2026-04-19` が別途作成される |
| 4 | パーサー出力の `bm.folder` が無視される | 全ブックマークが当日フォルダに入る（ソース由来のフォルダは作られない） |
| 5 | `handleImportComplete` 後に `currentFolder` が当日フォルダになっている | state 上で切り替わっていることを確認 |
| 6 | 重複 URL がスキップされる | 既存動作と同じ |

### 動作確認（手動）

- YouTube Watch Later CSV（140件）でインポート → リロードせずに `インポート 2026-04-18` フォルダに 140 件表示されること
- Chrome ブックマーク HTML（数フォルダ含む）でインポート → 全ブックマークが単一フォルダに平坦化されること
- 同じファイルを 2 回連続インポート → 2 回目は全件「重複スキップ」になること

---

## 非目標（この変更ではやらない）

- ソースフォルダ情報を description やタグに保存（将来の拡張として保留）
- ソース別インポートフォルダオプション（`YouTube インポート` 等）
- インポート履歴画面
- 汎用 CSV パーサー（他サービスからのエクスポートフォーマット互換）— 別スプリントで対応
