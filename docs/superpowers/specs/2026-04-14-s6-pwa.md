# S6: PWA + スマホ保存 設計書

> 承認済み: 2026-04-14

---

## 概要

Booklage を PWA（Progressive Web App）化し、スマホのホーム画面に追加・オフライン閲覧・共有メニューからの保存を可能にする。

---

## 1. PWA アイコン

### 必要なファイル

| ファイル | サイズ | 用途 |
|---------|--------|------|
| `public/icon-192.png` | 192x192 | ホーム画面アイコン、PWA インストール |
| `public/icon-512.png` | 512x512 | スプラッシュ画面、ストア表示 |

### デザイン

- Booklage のブランドカラー（紫 `#7c5cfc`）ベースの背景
- 白い「B」文字またはブックマーク+コラージュをイメージしたシンプルなアイコン
- 角丸なし（OS が自動的に角丸を適用するため）

### manifest.json

既に `public/manifest.json` にアイコン参照が設定済み。ファイルを配置するだけで動作する。

---

## 2. Service Worker（オフライン対応）

### 方針

手書きの軽量 Service Worker。Workbox 等のライブラリは使わない（キャッシュ対象が単純なため）。

### キャッシュ戦略

| 対象 | 戦略 | 理由 |
|------|------|------|
| HTML ページ（`/`, `/board`, `/save` 等） | Network First | 最新版を優先、オフライン時はキャッシュ |
| JS/CSS（`/_next/static/*`） | Cache First | ハッシュ付きファイル名で不変 |
| フォント（Google Fonts） | Cache First | 頻繁に変わらない |
| PWA アイコン・manifest | Cache First | 静的ファイル |
| API（`/api/*`） | Network Only | OGP 取得はオンライン必須 |

### ファイル構成

- `public/sw.js` — Service Worker 本体
- `app/layout.tsx` — SW 登録スクリプトを追加（`<script>` タグ）

### SW 登録

```typescript
// app/layout.tsx の <head> 内に追加
<script dangerouslySetInnerHTML={{ __html: `
  if ('serviceWorker' in navigator) {
    navigator.serviceWorker.register('/sw.js')
  }
`}} />
```

### キャッシュバージョニング

SW 内にバージョン文字列を持ち、デプロイ時に更新。古いキャッシュは `activate` イベントで自動削除。

### オフライン時の体験

- `/board` — IndexedDB のデータでカード表示可能。ドラッグ・スタイル切替も動作。
- URL 追加（OGP 取得）— オフラインでは失敗するが、エラーメッセージを表示。カードは URL のみで仮作成し、オンライン復帰後に OGP を取得する設計も将来検討。
- LP・静的ページ — キャッシュ済みなら表示可能。

## 3. Web Share Target（スマホからの保存）

### 仕組み

1. ユーザーがスマホで Booklage を PWA としてインストール
2. 任意のアプリ（Twitter, YouTube, Safari 等）で「共有」をタップ
3. 共有先に「Booklage」が表示される
4. タップすると Booklage のボード画面が開き、共有された URL が自動的にカードとして追加される

### manifest.json の share_target（設定済み）

```json
{
  "share_target": {
    "action": "/board?shared=true",
    "method": "GET",
    "params": {
      "title": "title",
      "text": "text",
      "url": "url"
    }
  }
}
```

### ボード画面での受け取り処理

`board-client.tsx` で URL パラメータを検出して処理:

1. ページロード時に `?shared=true` を検出
2. `url` パラメータから URL を取得（`text` からの URL 抽出もフォールバック）
3. OGP API を呼び出してカード情報を取得
4. IndexedDB に保存してカードを表示
5. URL パラメータをクリーンアップ（`history.replaceState`）

### URL の抽出

共有元アプリによって URL の渡し方が異なる:
- `url` パラメータに直接 URL が入る場合
- `text` パラメータに「タイトル URL」形式で入る場合
- `text` パラメータに URL のみ入る場合

`text` からの URL 抽出には正規表現を使用:
```typescript
function extractUrl(text: string): string | null {
  const match = text.match(/https?:\/\/[^\s]+/)
  return match ? match[0] : null
}
```

## 4. インストール促進 UI

### 方針

控えめな UI。強制ポップアップは使わない。

### 実装

- `beforeinstallprompt` イベントをキャッチして保存
- ボード画面の適切な場所に「アプリとして追加」ボタンを表示（初回のみ、または設定パネル内）
- ボタンクリックで `prompt()` を呼び出し
- インストール済みの場合はボタンを非表示

### コンポーネント

- `components/pwa/InstallPrompt.tsx` — インストール促進 UI（'use client'）

---

## 5. やらないこと

- プッシュ通知（ユーザーデータ非保持の設計と矛盾）
- バックグラウンド同期（サーバー DB なし）
- 強制インストールポップアップ
- Workbox ライブラリの導入（オーバーキル）
