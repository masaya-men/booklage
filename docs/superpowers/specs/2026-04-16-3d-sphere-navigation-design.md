# 3D球体ナビゲーション — 設計仕様書

> 作成日: 2026-04-16
> ステータス: 設計承認待ち

---

## 1. コンセプト

無限キャンバスを球面に変える。今の2Dキャンバスでは端のカードが「潰れる」（snapdom方式）問題を、**パンし続けると地球の裏側に回り込む** 球面ナビゲーションで解決する。

### 核心的な体験

- 中心付近はほぼ2D平面（今と同じ使い心地）
- 端に向かうとカードが球面的にカーブし、裏側に回り込む
- パンし続けると一周して元に戻る
- ズームインで平面的に、最大ズームアウトで球体全体が見える
- ユーザーが球面モード ↔ 2D平面モードを切替可能

### 理念

> 最先端の見た目、パフォーマンスも一切妥協なく240FPSを目指して、UIUXに極限までこだわったAIっぽくない最高のユーザー体験を提供する。

技術的な方法は変わっても、この理念は絶対にブレない。

## 2. 技術構成: WebGL + CSS3D ハイブリッド

Three.jsの2つのレンダラーを同じシーンで同時に使う（Three.js公式テクニック）。

### 下層: WebGLRenderer

- **球体ジオメトリ** — 本物の3D球面
- **ガラスシェーダー** — 物理ベース屈折（参考サイト級）
- **ライティング** — 環境光・反射
- **ポストプロセス** — ブルーム・被写界深度・ビネット
- **裏側カードのドット表現** — WebGLポイントスプライト

### 上層: CSS3DRenderer

- **カードDOM要素** — react-tweet、YouTube oEmbed、OGPカード全てそのまま
- **クリック・ドラッグ** — DOM操作がそのまま動作
- **CSSアニメーション** — float、springホバー等そのまま

### 追加依存

- `three` — WebGLRenderer + CSS3DRenderer + ポストプロセス（tree-shaking後 約80KB gzip）

## 3. リキッドグラス仕様

### 方針

- **透明度最優先** — ガラス自体の色はほぼゼロ。背景が完全に透けて見える
- **「色」ではなく「歪み」でガラスを表現** — 不透明な白やブルーのガラスではなく、レンズのような屈折効果
- **屈折の歪み（ディストーション）を大きく** — 参考サイトのように、ガラス越しに背景がぐにゃっと歪む

### シェーダーエフェクト

- **物理ベース屈折（IOR）** — 背景が歪んで見えるリアルなガラス
- **フレネル反射** — 見る角度で反射率が変わる
- **環境マッピング** — 周囲が映り込む
- **コースティクス** — ガラスを通った光が作る集束模様
- **色収差** — RGBがわずかにズレてプリズム効果

## 4. 操作モデル

### 球面回転（パン）— 3つの方法を並列提供

1. **空きスペースをドラッグ** — 最も直感的。初心者向け
2. **Space + ドラッグ** — カードの上でも効く。パワーユーザー向け
3. **右クリック長押し + ドラッグ** — カード上でも効く。もう1つの選択肢

全て同じ球面回転操作。慣性スクロール付き（GSAP easing）。

### ズーム

- ホイール / ピンチ（今と同じ操作）
- ズームイン → 球面が平らになる（Google Maps的）
- ズームアウト → 球面のカーブが見えてくる
- 最大ズームアウト → 球体全体が画面に収まる俯瞰ビュー

### カードインタラクション（3D空間内）

- **クリック** → 3Dフリップ → URL遷移（今と同じ）
- **ホバー** → スプリング物理 + ツールチップ（今と同じ）
- **ドラッグ移動** → 球面上の座標で移動（カードは球面に沿って動く）
- **リサイズ** → 正面に近いカードのみ対応

---

## 5. カードの球面上の見え方（段階的変化）

| 段階 | 視野角からの距離 | 表示方法 | 描画コスト |
|------|-----------------|---------|-----------|
| 正面 | 0°〜45° | フルDOM（全機能） | 高 |
| 斜め | 45°〜75° | DOM + 3D遠近法で傾き + ブラー | 中 |
| 裏側手前 | 75°〜120° | ドット（WebGLポイントスプライト） | 極小 |
| 完全裏側 | 120°〜180° | 非表示 | ゼロ |

段階間はスムーズに遷移する（急に切り替わらない）。

---

## 6. 球体サイズ

カード枚数に応じて球の半径を自動調整。

- カードが少ない → 小さな球体（密度を保つ）
- カードが多い → 大きな球体（ゆったり配置）
- 半径 = f(カード枚数) で動的計算
- 常に適切な密度を維持

## 7. パフォーマンス戦略（240FPS目標）

### ターゲット

- カード100枚: 240FPS（必達）
- カード1000枚: 240FPS（目標）
- カード5000枚: 240FPS（究極目標）

### 球面版ビューポートカリング

- 正面（視野角内）→ フルDOM描画 + CSS3DRenderer
- 斜め（視野角の端）→ DOM描画あり、ポストプロセスでブラー
- 裏側手前 → DOMを非表示、WebGLパーティクル（ドット）に置き換え
- 完全裏側 → 何も描画しない

### LOD（Level of Detail）

- ズームイン: カード全要素表示（画像、テキスト、react-tweet等）
- 中距離: サムネイル + タイトルのみ
- ズームアウト: カラードット（フォルダの色で）
- 最大ズームアウト: WebGLポイントスプライト（超軽量、数千個OK）

### 最適化手法

- 見えないカードのDOMは完全にアンマウント
- 球面回転中はカード内のリッチコンテンツ（iframe等）を一時停止
- `will-change: transform` でGPUレイヤー化
- ガラスシェーダーは正面のUIパネルにのみ適用
- ポストプロセスは1パスでまとめる

## 8. モード切替とフォールバック

### 球面モード ↔ 2D平面モード

- ツールバーに切替ボタン（🌐 Globe / 📋 Flat）
- 切替アニメーション: 球面が滑らかに平面に展開（GSAP Timeline、約0.8秒）
- 2D平面モードは今のキャンバスそのまま（既存機能を壊さない）
- ユーザーの選択はIndexedDBに保存、次回起動時に復元
- グリッド ↔ コラージュ切替時は前回配置を保持しない（初回切替時に注意表示）

### 初回体験

- 球面モードをデフォルトで表示
- 操作ヒント: 「ドラッグで回す」「ピンチでズーム」「最大ズームアウトで全体像」

### WebGL非対応フォールバック

- WebGLが使えない場合は自動的に2D平面モードのみ
- 切替ボタンを非表示、今のキャンバスがそのまま動作

---

## 9. 球体の視覚表現

- 球体自体は「見えない」— カードの配置と端のカーブで球面を感じさせる
- ズームアウト時のみ、球体の輪郭がうっすら光るリング（大気のグロー効果）で存在を示す
- 背景は現在のテーマ（カッティングマット等）がそのまま

### ポストプロセス

- **ブルーム** — 光源やガラスの反射が柔らかく光る
- **被写界深度（DOF）** — 端のカードが自然にぼける
- **ビネット** — 画面端が少し暗くなり没入感アップ

## 10. ファイル構成

```
lib/
  sphere/
    sphere-projection.ts    — 2D座標 ↔ 球面座標の変換数学
    sphere-renderer.ts      — Three.js WebGL + CSS3D セットアップ
    glass-shader.ts         — 物理ベース屈折ガラスシェーダー
    post-processing.ts      — ブルーム・DOF・ビネット
    sphere-culling.ts       — 球面版ビューポートカリング + LOD
    sphere-interaction.ts   — 球面上のドラッグ・回転・慣性
  canvas/
    use-infinite-canvas.ts  — 既存を拡張（モード切替対応）
components/
  board/
    SphereCanvas.tsx        — 球面モードのキャンバスコンポーネント
    Canvas.tsx              — 既存（2D平面モード、変更最小限）
    board-client.tsx        — 既存を拡張（モード切替ロジック追加）
    ViewModeToggle.tsx      — 🌐/📋 切替ボタン
```

### 既存コードへの影響

- `Canvas.tsx` — ほぼそのまま。2D平面モードとして残る
- `board-client.tsx` — モード切替ロジック追加。カードデータの受け渡し
- `use-infinite-canvas.ts` — 球面モード時の座標変換を追加
- カードコンポーネント群 — **変更なし**

### データモデル

- カードの位置は引き続き2D座標（x, y）でIndexedDBに保存
- 球面表示時は2D→球面座標にリアルタイム変換
- モード切替しても位置データが壊れない
