# セッションワークフロー

## 開始時
1. `docs/TODO.md` を読む — これだけで現在の状態・タスク・注意事項がわかる
2. 他のドキュメントは該当タスクに着手するときに読む

## 作業中
- 会話で決まったアイデア・方針・設計判断は即座に `docs/TODO.md` に追記する
- 完了タスクは `docs/TODO_COMPLETED.md` に移動（本当に完了しているかコードで確認してから）
- TODO.mdは常に50行以内を目標に保つ

## 終了時
1. TODO.md先頭の「現在の状態」セクションを更新
2. **`docs/private/dashboard.html` (= 永続ダッシュボード、 ユーザー大画面閲覧用) を最新化** ← セッション中に変化した内容を全て反映:
   - 着手済 → ✅ 完了 dot に更新 (= candidate / item のステータス dot)
   - 新着アイデアは「新着」 パネル (= 🌸 pink dot) に追加 + テーマ案に該当があれば該当 panel にも追加
   - TODO.md / IDEAS.md と齟齬がないか確認 (= dashboard は両ファイルの「視覚レンダリング」、 真実の場所は md)
   - 「今のゴール」 hero strip を次セッションのものに上書き
3. コミット → push までセットで完了
4. ユーザーに引き継ぎメッセージを出力（次セッションの最初にコピペできる形式）
   - 変更したファイル一覧、次の最優先タスク、docs/TODO.md 読め指示、 ダッシュボードリンク

## コンパクション時の保持指示
When compacting, always preserve: 現在のタスク、変更中のファイルパス、TODO.mdの「現在の状態」セクション内容
